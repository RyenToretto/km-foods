# 改良菜谱

## 指令

读取指定菜谱，从多个方向提出改良建议。

### 分析维度

对每个维度，给出具体可操作的建议：

1. **更健康** — 减盐减油减糖、替换更健康的食材、增加膳食纤维
2. **更简单** — 减少步骤、用更易得的食材替代、缩短烹饪时间
3. **更美味** — 提升口感的技巧、增加风味层次、改善呈现
4. **更经济** — 用平价食材替代昂贵食材、减少浪费
5. **更适合新手** — 降低技术难度、提供更详细的判断标准

### 输出格式

```markdown
## 改良建议：{菜名}

### 更健康
- ...

### 更简单
- ...

### 更美味
- ...

### 更经济
- ...

### 更适合新手
- ...

### 综合推荐改动
1. {最值得做的改动}
2. {次要改动}
```

如果用户指定了某个方向（如"我想做得更健康"），优先详细展开该方向。
