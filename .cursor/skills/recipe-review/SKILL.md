# 审查菜谱

## 指令

对指定菜谱（或所有菜谱）进行格式和内容审查。

### 第 1 步：读取模板和规则

- 读取 `templates/recipe-template.md` 获取标准格式
- 参照 `.cursor/rules/recipe-format.mdc` 中的格式要求

### 第 2 步：格式检查

逐项检查以下必要章节是否存在且格式正确：

- [ ] 一级标题（菜名）
- [ ] 描述引用块
- [ ] 基本信息表格（含分类、口味、难度、用时、份量）
- [ ] 食材表格（含用量列）
- [ ] 调料表格（含用量列）
- [ ] 用具列表
- [ ] 做法（含准备工作、烹饪步骤、装盘）
- [ ] 技巧与窍门

### 第 3 步：内容质量检查

- 用量是否具体（标记只写"适量"的条目）
- 做法步骤是否包含火候和时间信息
- 技巧是否具有实际参考价值

### 第 4 步：输出审查报告

使用三级反馈分类：

```markdown
## 审查报告：{菜名}

### 必须修复
- {严重的格式缺失或错误}

### 建议改进
- {可以提升质量的修改}

### 锦上添花
- {可选的优化点}

### 总评
{通过 / 需修改后通过}
```

批量审查时，先输出总览表格，再逐个展开问题菜谱的详情。
